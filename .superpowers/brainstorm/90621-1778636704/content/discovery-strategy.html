🔍 目录发现策略 — 硬编码 vs 动态扫描

你的目录会变，APP 如何适应？

策略对比
❌ 纯硬编码（不可取）
const knownDirs = ['~/.npm', '~/.cache', '~/.docker', '~/.kube', ...]
// 只认这些目录，其他全部忽略

问题：换了电脑、装了新工具就失效

⚠️ 纯动态扫描（太激进）
// 扫描所有 ~/.xxx 目录，按大小排序
// ~/.ssh、~/.config、~/.zshrc 全部列出来
// 结果：列出 50+ 目录，用户不知道该删哪个

问题：列出太多无关目录，误删风险高

✅ 混合策略（推荐）
// 1. 预设「已知目录库」— 能识别的都给友好名称和分类
// 2. 动态扫描 — 发现未识别的 ~/.xxx 目录
// 3. 智能推断 — 按命名模式推测归属
混合策略 — 工作流程
步骤 1: 扫描所有 ~/.xxx 目录 
find ~ -maxdepth 1 -name '.*' -type d
→ 找到 33 个隐藏目录
步骤 2: 匹配已知目录库 
~/.npm     → ✅ 匹配 "npm cache" (可清理缓存)
~/.docker  → ✅ 匹配 "Docker 配置" (配置类)
~/.claude  → ✅ 匹配 "Claude Code" (工具数据)
~/.foobar  → ❌ 未匹配
~/.bazqux  → ❌ 未匹配
步骤 3: 智能推断未匹配目录 
~/.foobar  → 检查 /Applications 是否有 Foobar.app
          → 检查 ~/Library 是否有 com.foobar.*
          → 检查内部结构: 有 node_modules? → 推断为 Node 工具
          → 有 cache/ 子目录? → 推断为缓存
          → 有 config.json? → 推断为配置
步骤 4: 分类展示 
🟢 已知可清理: ~/.npm (1.2GB) — "npm 包缓存"
🟡 已知需确认: ~/.docker (3.4MB) — "Docker 配置"
🟠 未知但可清理: ~/.foobar (50MB) — "推断为缓存目录"
⚪ 未知待分类: ~/.bazqux (12MB) — "未识别，请用户确认"

核心思想

已知目录库是「字典」，不是「白名单」。

字典的作用是：

  • 已知目录 → 给友好名称 + 明确分类 + 一键清理
  • 未知目录 → 动态发现 + 智能推断 + 标记为「待分类」让用户决定
  • 字典会持续更新：用户发现新目录 → 贡献到字典 → 所有人受益

文档里列的目录只是"当前字典"，不是最终列表。
